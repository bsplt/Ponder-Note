  <behavior>
GFM task list checkbox patterns: `[ ]`, `[x]`, `[X]` with optional leading list markers (-, *, +).

**Test Cases:**

1. **Basic unchecked checkbox:**
   - Input: stem="test", body="- [ ] buy milk"
   - Output: TodoItem { text: "buy milk", checked: false, note_stem: "test", line_number: 0, char_offset: 2 }

2. **Basic checked checkbox:**
   - Input: stem="test", body="- [x] buy eggs"
   - Output: TodoItem { text: "buy eggs", checked: true, note_stem: "test", line_number: 0, char_offset: 2 }

3. **Uppercase checked:**
   - Input: stem="test", body="- [X] buy bread"
   - Output: TodoItem { text: "buy bread", checked: true, note_stem: "test", line_number: 0, char_offset: 2 }

4. **No list marker:**
   - Input: stem="test", body="[ ] standalone checkbox"
   - Output: TodoItem { text: "standalone checkbox", checked: false, note_stem: "test", line_number: 0, char_offset: 0 }

5. **Leading whitespace:**
   - Input: stem="test", body="  - [ ] indented task"
   - Output: TodoItem { text: "indented task", checked: false, note_stem: "test", line_number: 0, char_offset: 4 }

6. **Skip fenced code blocks:**
   - Input: "```\n- [ ] not a todo\n```\n- [ ] real todo"
   - Output: Only extract "real todo" (line 3)

7. **Skip blockquotes:**
   - Input: "> - [ ] quoted\n- [ ] real todo"
   - Output: Only extract "real todo" (line 1)

8. **Multiple todos:**
   - Input: "- [ ] first\n- [x] second\n- [ ] third"
   - Output: 3 TodoItem structs with correct line numbers (0, 1, 2)

9. **Character offset precision:**
   - Input: "    - [ ] deeply indented"
   - Output: char_offset: 6 (4 spaces + "- ")

10. **Empty note:**
    - Input: ""
    - Output: empty Vec

**Regex pattern (from research):**
```
^(?P<indent>\s*)(?:[-*+]\s+)?(?P<checkbox>\[(?P<state>[ xX])\])\s+(?P<text>.+)$
```

**TodoItem struct:**
```rust
pub struct TodoItem {
    pub text: String,
    pub checked: bool,
    pub note_stem: String,    // Note stem for identifying source note
    pub line_number: usize,
    pub char_offset: usize,  // Position of '[' in line
}
```

**Fenced block detection (follow note_rewrite.rs pattern):**
- Track in_fence boolean
- Toggle on lines starting with ``` or ~~~
- Skip checkbox extraction while in_fence == true
