<task type="checkpoint:human-verify" gate="blocking">
  <what-built>
Complete todo workflow: extraction from markdown, browsable list grouped by tags, in-place checkbox toggling with optimistic updates, and keyboard navigation.

Backend: Todo extraction (domain/todo.rs), safe toggle with verification, Tauri commands (list_todos, toggle_todo)
Frontend: TodoList screen, grouping utility, App routing with T key, optimistic updates with rollback
  </what-built>
  <how-to-verify>
**Setup:**
1. Start the app: `cd app && npm run tauri dev`
2. Create test notes with todos:
   - Note A (tags: work, urgent): Add 2-3 unchecked todos `- [ ] task text`
   - Note B (tags: work): Add 1-2 checked todos `- [x] done task`
   - Note C (tags: personal): Add 1 unchecked todo
   - Note D (no tags): Add 1 unchecked todo

**Test Cases:**

1. **Open Todo List:**
   - From Overview, press `T`
   - Expected: Todo List screen opens, shows all todos

2. **Verify grouping:**
   - Expected: Todos grouped by tag headers ("work", "urgent", "personal", "Untagged")
   - Expected: Note A's todos appear in BOTH "work" AND "urgent" groups (cross-group appearance)
   - Expected: "Untagged" group appears last

3. **Verify ordering:**
   - Edit Note B (make it most recent)
   - Exit and press T again
   - Expected: "work" group appears first (most recent activity)
   - Expected: Within "work" group, Note B's todos appear before Note A's todos

4. **Toggle with mouse:**
   - Click on an unchecked todo row
   - Expected: Checkbox instantly changes to [x], text dims
   - Expected: All instances in other groups also update immediately

5. **Toggle with keyboard:**
   - Press Arrow Down to navigate to a todo
   - Press Space
   - Expected: Checkbox toggles, visual feedback instant
   - Expected: Focus indicator (border) stays on same row

6. **Open source note:**
   - Click the → button on a todo row (choose a todo NOT at the top of its note)
   - Expected: Editor opens with that note's content
   - Expected: Editor cursor/view is at line 0 (top of note), NOT at the todo line number (locked decision: "Opens at top of note")
   - Press Escape to return to Overview, press T to return to Todo List

7. **Toggle rollback on error:**
   - Delete a note's markdown file externally while Todo List is open
   - Try to toggle a todo from that note
   - Expected: Toast error appears, checkbox reverts to original state

8. **Keyboard navigation:**
   - Press Arrow Up/Down
   - Expected: Focus moves between todos (across groups seamlessly)
   - Press Enter on focused todo
   - Expected: Opens source note in Editor

9. **Exit Todo List:**
   - Press Escape
   - Expected: Returns to Overview

10. **Completed todo visibility:**
    - Toggle a todo to checked
    - Expected: Todo stays visible with [x] checkbox
    - Press Escape, press T again
    - Expected: Checked todo no longer appears (only shows open todos by default)

11. **Edge cases:**
    - Create note with nested checkboxes (indented)
    - Expected: All checkboxes appear as flat list (nesting not preserved visually)
    - Create note with checkbox in fenced code block: \`\`\`\n- [ ] code\n\`\`\`
    - Expected: Checkbox in code block NOT extracted
    - Create note with checkbox in blockquote: > - [ ] quoted
    - Expected: Quoted checkbox NOT extracted

**Performance check:**
- With 50+ todos across multiple notes, verify:
  - Todo List loads in < 1 second
  - Keyboard navigation feels instant
  - Toggle feedback is immediate (no lag)

**Visual design check:**
- Todo List matches Overview design language (spacing, colors, typography)
- Focus indicator is clear and consistent with Overview
- Completed todos are visually distinct but not distracting
- Tag group headers are readable but subtle
  </how-to-verify>
