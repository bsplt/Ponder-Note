Tauri app scaffold + Rust WorkspaceService + Workspaces/Overview UI implementing workspace assignment, switching, persistence, and root-only note discovery.
  </what-built>
  <how-to-verify>
1) Start the app: `nix develop -c pnpm -C app tauri dev`.
2) Create a test workspace folder `~/tmp/ponder-ws-a` with:
   - `a.md` in the root
   - `.ponder/hidden.md`
   - `deleted/deleted.md`
3) In the app, assign `~/tmp/ponder-ws-a` to slot 1 via the folder picker.
   Expected:
   - Slot 1 becomes active immediately.
   - Overview shows notes list containing `a.md`.
   - Overview does NOT show `hidden.md` or `deleted.md`.
4) Create a second workspace `~/tmp/ponder-ws-b` with root `b.md` and assign it to slot 2.
5) From Overview, press `1` and `2` to switch.
   Expected:
   - Active workspace indicator updates.
   - Notes list updates to `a.md` vs `b.md`.
   - A confirmation toast appears on switch.
6) Relaunch persistence:
   - Quit the dev app window.
   - Start `pnpm -C app tauri dev` again.
   Expected:
   - App opens with the last active slot restored.
7) Missing folder recovery:
   - Set active slot to one pointing at a folder you delete/rename on disk.
   - Relaunch.
   Expected:
   - App routes to Workspaces.
   - The problematic active slot shows an error state.
   - A fallback assigned valid slot is highlighted (if any exist).
8) Keyboard safety:
   - Focus a text input (if Overview has none, temporarily add one for this test).
   - Press `1`-`9`.
   Expected:
   - No workspace switching occurs while typing.
