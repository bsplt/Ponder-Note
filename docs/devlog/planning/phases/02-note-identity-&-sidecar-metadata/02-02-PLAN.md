<task type="auto">
  <name>Task 2: Implement sidecar rebuild + note listing/creation in workspace service</name>
  <files>app/src-tauri/src/services/workspace_service.rs</files>
  <action>
Extend `WorkspaceService` with:
- `list_notes()` that:
  - Resolves the active workspace path and validates it (same checks as list_root_notes).
  - Iterates root `*.md` files only; parse the filename stem as UNIX ms (skip if parse fails).
  - Reads the first line, derives title via `derive_note_title`.
  - Ensures `.ponder/meta/` exists and reads sidecar at `.ponder/meta/<stem>.json`.
  - If sidecar is missing or invalid JSON, rebuild it with `title` + `created_at` from filename, preserving unknown fields when possible.
  - If sidecar exists but `created_at` conflicts with filename, rewrite with filename value.
  - If title differs from derived title, update sidecar title (keep `updated_at`/`tags` as-is).
  - Return `NoteSummary` with `tags` defaulting to empty when absent.
- `create_note()` that:
  - Generates a UNIX ms timestamp filename and writes `<timestamp>.md` in workspace root (empty contents is ok).
  - Writes sidecar immediately with `title: "Untitled"` and `created_at` set to filename timestamp.
  - Returns the created `NoteSummary`.
Handle IO/serde errors via new `WorkspaceServiceError` variants and map them to stable error codes.
Never derive notes from sidecars; ignore orphan sidecars entirely.
  </action>
  <verify>nix develop -c cargo check</verify>
  <done>Backend can list notes with rebuilt sidecars and create new timestamp-named notes</done>
</task>
