<tasks>

<task type="checkpoint:human-verify" gate="blocking">
  <what-built>
Complete search and tags functionality:
- Tag pill badges in overview note rows (sorted alphabetically)
- SearchBar with tag filter pills and text input
- Real-time note filtering by search text (with wildcards) and tag filters
- Tag editing in editor with pill input and autocomplete
- Tag persistence to sidecar JSON on blur
  </what-built>
  <how-to-verify>
**Test 1: Tag Display**
1. Launch the app and open a workspace with notes
2. If no notes have tags, create a note and add tags (see Test 3 below)
3. Return to overview and verify:
   - Notes with tags show pill badges on the right
   - Tags are sorted alphabetically
   - Tags have subtle muted styling (rounded, small, gray background)

**Test 2: Search Functionality**
1. In overview, press Cmd+F to focus search input
2. Type a partial note title (e.g., "meet")
3. Verify:
   - Note list filters to show only matching notes
   - Filtering happens real-time as you type (no Enter needed)
4. Try wildcard search (e.g., "meet*ing")
5. Try multi-word search (e.g., "project meeting")
6. Verify:
   - Wildcard matches work
   - Multi-word uses AND logic (both words must appear)
7. Clear search and verify all notes return

**Test 3: Tag Filtering**
1. Click a tag badge in a note row
2. Verify:
   - Tag appears as filter pill in search bar
   - Note list filters to show only notes with that tag
3. Click another tag (in a different note)
4. Verify:
   - Second tag pill appears in search bar
   - Note list shows only notes with BOTH tags (AND logic)
5. Click X on a filter pill
6. Verify pill is removed and filtering updates
7. Try Cmd+click on a tag to exclude it
8. Verify:
   - Tag appears as red pill with "-" prefix
   - Notes with that tag are hidden
9. Clear all filter pills (click X or backspace)

**Test 4: Search + Tag Combination**
1. Add a tag filter pill (click a tag)
2. Type search text in the search bar
3. Verify:
   - Note list shows notes matching BOTH tag filter AND search text
   - Search text input works normally alongside filter pills

**Test 5: Tag Editing**
1. Open a note in the editor
2. Verify tag input appears at top (above note content)
3. Type a tag name and press Enter or Comma
4. Verify:
   - Tag becomes a pill
   - Placeholder changes to "+"
5. Type a few characters and verify autocomplete dropdown appears
6. Verify autocomplete shows existing workspace tags
7. Use arrow keys to navigate suggestions, press Tab to accept
8. Add multiple tags, then click X on a pill to remove it
9. Click outside the tag input (blur)
10. Wait a moment, then return to overview
11. Verify:
    - Edited tags appear on the note row in overview
    - Tags are sorted alphabetically

**Test 6: Search Persistence**
1. In overview, type search text and add tag filter
2. Open a note (double-click)
3. Press ESC to return to overview
4. Verify:
   - Search text and tag filters are still active
   - Note list still shows filtered results

**Test 7: Keyboard Shortcuts**
1. In overview, press Cmd+F
2. Verify search input receives focus
3. Add filter pills, then press Backspace with empty search text
4. Verify last filter pill is removed
5. Type `#work` in search bar
6. Verify it creates a tag filter pill (basic version)

**Expected Outcomes:**
- All tag badges render correctly and are sorted alphabetically
- Search filters notes in real-time with wildcard support
- Tag filters work via click and combine with AND logic
- Exclude filters work via Cmd+click
- Tag editing persists to backend and appears in overview
- Autocomplete suggests existing workspace tags
- Search state persists across editor navigation
- Keyboard shortcuts (Cmd+F, Backspace, Enter, Comma, Tab) work
  </how-to-verify>

  </resume-signal>
