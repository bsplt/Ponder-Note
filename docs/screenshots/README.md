# README Screenshots

Store README screenshots in this folder.

## Recommended conventions

- Prefer `webp` for UI screenshots (`png` only if needed for quality).
- Keep filenames lowercase and descriptive, for example:
  - `overview-main.webp`
  - `editor-markdown-preview.webp`
  - `todos-grouped-by-tag.webp`
- Use one scale consistently (for example 2x Retina captures).
- Keep each file reasonably small (target: under 500 KB when possible).
- Avoid sensitive data in screenshots.

## Markdown usage

Use repository-relative links in `README.md`, for example:

```md
![Overview screen](docs/screenshots/overview-main.webp)
```
