  <name>Task 1: Create search and filter utilities</name>
  <files>
app/src/utils/search.ts

   <action>
Create `app/src/utils/search.ts` with the following utilities:

1. `escapeRegExp(str: string): string` — escapes regex special characters EXCEPT `*`
   - Split on `*`, escape each part with `str.replace(/[.*+?^${}()|[\]\\]/g, '\\$&')`, join with `.*`
   - Example: "meet*ing" becomes "meet.*ing"

2. `buildSearchRegex(query: string): RegExp | null` — builds case-insensitive AND-logic regex
   - If query is empty/whitespace, return null
   - Split query on whitespace into terms
   - For each term, escape and convert `*` to `.*`
   - Build lookahead pattern: `(?=.*term1)(?=.*term2)` for AND logic
   - Return new RegExp(pattern, 'i')
   - Example: "project meeting" matches notes with both words anywhere

3. `filterNotes(notes: NoteSummary[], searchText: string, includeTags: string[], excludeTags: string[]): NoteSummary[]`
   - Filter by includeTags first (AND logic): `includeTags.every(tag => note.tags?.includes(tag))`
   - Filter by excludeTags: `!excludeTags.some(tag => note.tags?.includes(tag))`
   - Filter by searchText using buildSearchRegex against `note.title` only
   - Return filtered array in original order (chronological by timestamp)

**IMPORTANT — Title-only search for Phase 4:**
CONTEXT.md specifies "search covers title + body", but NoteSummary type (from workspace API) currently only includes `stem`, `title`, and `tags` — no body field. To implement body search, we would need to:
- Load full note content for all notes in workspace (expensive)
- Add body field to NoteSummary type
- Update workspace_list_notes backend command to include body preview or full content

For Phase 4, implement **title-only search** as specified above. This satisfies OV-04's core filtering requirement. Body search is a future enhancement that requires backend changes to include note content in NoteSummary.

Import NoteSummary type from '../api/workspace'.

Follow RESEARCH.md patterns for wildcard regex construction and filtering logic.
