  void (async () => {
    if (!stem) return

      await workspaceUpdateNoteTags(stem, tags)
      setTagSaveError(null)

NOTE: Do not debounce tag autosave. CONTEXT.md specifies "autosave on blur" (when user leaves field), not continuous autosave like the note content textarea. This is intentional to avoid excessive writes for each tag pill added/removed.

**CRITICAL: Trigger workspace refresh after tag save**
After tag save succeeds, dispatch 'workspace-refresh-needed' event so Overview can refresh the note list and display updated tags immediately. Without this, tags won't appear in overview until user manually returns to overview or refreshes the app.

Add to Overview.tsx to listen for refresh events:
```typescript
useEffect(() => {
  const handleRefresh = () => {
    // Re-fetch workspace notes to get updated tags
    void loadWorkspaceNotes()
  }
  window.addEventListener('workspace-refresh-needed', handleRefresh)
  return () => window.removeEventListener('workspace-refresh-needed', handleRefresh)
}, [loadWorkspaceNotes])
```
