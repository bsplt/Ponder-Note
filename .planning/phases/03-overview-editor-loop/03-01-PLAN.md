<feature>
  <name>Exit checklist rewrite</name>
  <files>app/src-tauri/src/domain/note_rewrite.rs</files>
  <behavior>
    Convert lines that start with optional indentation + `o ` or `O ` into `[ ] `, preserving indentation.
    Skip conversion inside fenced code blocks (``` or ~~~ toggles) and for blockquote lines (`>` after optional indentation).
    Only rewrite when there is text after `o `; do not match tabs or `o` with no trailing content.
    Cases: "o task" -> "[ ] task", "  O task" -> "  [ ] task", "o " -> unchanged, "> o task" -> unchanged, fenced blocks unchanged.
  </behavior>
  <implementation>
    Add `rewrite_exit_checklists(body: &str) -> String` (or similar) to a new module.
    Track fenced code block state; treat lines starting with ``` or ~~~ (after optional indentation) as fence toggles.
    Apply case-insensitive `o ` rewrite only when not in a fence and not a blockquote line.
  </implementation>
</feature>
