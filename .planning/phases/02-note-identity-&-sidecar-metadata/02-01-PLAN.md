  <name>Task 1: Add note title derivation helper</name>
  <files>app/src-tauri/src/domain/note_title.rs</files>
  <action>
Create a new module that exposes `fn derive_note_title(first_line: &str) -> String` implementing Phase 2 rules:
- Convert tabs to spaces, then trim leading/trailing whitespace.
- Strip all leading `#` after optional leading spaces.
- If result is empty/whitespace, marker-only (`#`, `###`), or `---`, return `Untitled`.
- Strip list/quote prefixes (`- `, `* `, `1. `, `> `) and checkbox prefixes (`- [ ]`, `- [x]`).
- Strip inline markdown formatting markers (emphasis/code) while preserving text content.
- For markdown links, keep link text only.
- Keep trailing `#` markers as part of the title.
Avoid regex-heavy parsing that could strip valid characters; keep rules ordered and deterministic.
  </action>
  <verify>nix develop -c cargo test</verify>
  <done>derive_note_title returns expected strings for representative inputs</done>
</task>

<task type="auto">
  <name>Task 2: Add unit tests for title normalization</name>
  <files>app/src-tauri/src/domain/note_title.rs</files>
  <action>
Add a focused `#[cfg(test)]` module with table-driven tests that cover:
- Heading markers: "# Title" -> "Title", "  ### Title" -> "Title".
- Marker-only: "#" and "###" -> "Untitled".
- Horizontal rule: "---" -> "Untitled".
- List/quote: "- Item" -> "Item", "1. Item" -> "Item", "> Quote" -> "Quote".
- Checkbox: "- [ ] Task" -> "Task", "- [x] Done" -> "Done".
- Inline formatting: "**Bold**" -> "Bold", "`Code`" -> "Code", "_Em_" -> "Em".
- Markdown links: "[Link](https://example.com)" -> "Link".
- Tabs/whitespace trimming: "\t\t Title  " -> "Title".
- Keep trailing hashes: "Title ###" -> "Title ###".
Ensure tests call derive_note_title directly and assert exact string outputs.
