  <name>Task 1: Implement a small workspace store + app boot flow</name>
  <files>
app/src/stores/workspaceStore.ts
app/src/App.tsx
  </files>
  <action>
- Create `app/src/stores/workspaceStore.ts` (no external state library):
  - Store shape: `slots` (1-9 with optional path), `activeSlot`, `activeStatus` (ok|unassigned|missing|unreadable), `fallbackSlot`, `loading`, `errorMessage`.
  - Actions:
    - `boot()` calls `workspace_get_state()`.
    - `assignSlot(slot, path)` calls `workspace_assign_slot()`.
    - `switchSlot(slot)` calls `workspace_switch_slot()` and then refreshes notes.
    - `refreshNotes()` calls `workspace_list_root_notes()`.
- In `app/src/App.tsx`:
  - On first mount, call `boot()`.
  - If boot resolves to `activeStatus === 'ok'`, default to `screen='overview'`.
  - If boot resolves to `unassigned` or `missing/unreadable`, set `screen='workspaces'`.
  - Keep routing simple: `screen` local state, but driven by store status.

Guardrails:
- No renderer filesystem reads.
- Do not validate non-active slots on boot; reflect only what Rust returned.
  </action>
  <verify>
- Run `nix develop -c pnpm -C app tauri dev` and confirm the app routes to Workspaces when no slot is assigned.
  </verify>
  <done>
- App boot calls `workspace_get_state()` and routes to Overview or Workspaces per the Phase 01 relaunch/missing-folder decisions.
  </done>
